# Java常见编程报错及解决方案汇总

## 一、空指针异常（NullPointerException）

**报错等级**：最高频运行时异常，新手头号杀手 

**报错原因**：对`null`对象调用了属性或方法 

**常见场景**： 

1. 字符串对象为null，调用equals()、length()等方法 
2. 数组为null，访问数组长度 
3. 自定义对象未初始化，调用对象方法 
4. 集合元素为null，调用元素方法 

**解决方案**：

```java
// 错误示例
String str = null;
if (str.equals("test")) { // 直接报错
    System.out.println("匹配");
}

// 正确示例1：先判空
String str = null;
if (str != null && str.equals("test")) {
    System.out.println("匹配");
}

// 正确示例2：常量放前面，避免空指针
if ("test".equals(str)) {
    System.out.println("匹配");
}

// 正确示例3：使用Optional（Java8+）
Optional.ofNullable(str).ifPresent(s -> System.out.println(s));
```

## 二、数组下标越界异常（ArrayIndexOutOfBoundsException）

**报错原因**：访问的数组下标超出了数组的有效范围

**常见场景**：

1. 数组下标从 0 开始，最大下标为`length-1`，访问`arr[arr.length]`直接报错
2. 循环条件错误，导致下标越界
3. 空数组访问下标

**解决方案:**

```java
// 错误示例
int[] arr = {1,2,3};
System.out.println(arr[3]); // 数组长度3，最大下标2

// 正确示例：循环边界控制
int[] arr = {1,2,3};
for (int i = 0; i < arr.length; i++) {
    System.out.println(arr[i]);
}

// 安全访问：先判断下标合法性
int index = 3;
if (index >=0 && index < arr.length) {
    System.out.println(arr[index]);
}
```

## 三、类型转换异常（ClassCastException）

**报错原因**：强制类型转换时，对象类型与目标类型不兼容

**常见场景**：

1. 父类对象强制转换为子类类型
2. 无继承关系的两个类互相转换
3. 集合泛型擦除导致的类型转换错误

**解决方案:**

```java
// 错误示例
Object obj = new Object();
String str = (String) obj; // 无继承关系，直接报错

// 正确示例：先使用instanceof判断类型
Object obj = "hello";
if (obj instanceof String) {
    String str = (String) obj;
    System.out.println(str);
}

// 最佳实践：使用泛型避免类型转换
List<String> list = new ArrayList<>();
list.add("test");
String str = list.get(0); // 无需强制转换
```

## 四、字符串下标越界异常（StringIndexOutOfBoundsException）

**报错原因**：调用 String 相关方法时，下标超出字符串长度范围

**常见场景**：

1. charAt (index) 方法，index 超出字符串长度
2. substring (start, end) 方法，start/end 超出范围
3. 空字符串调用下标相关方法

**解决方案：**

```java
// 错误示例
String str = "java";
System.out.println(str.charAt(4)); // 字符串长度4，最大下标3

// 正确示例：先判断长度与下标合法性
String str = "java";
int index = 4;
if (str != null && index >=0 && index < str.length()) {
    System.out.println(str.charAt(index));
}

// substring安全使用
String str = "hello";
if (str.length() >= 5) {
    System.out.println(str.substring(0,5));
}
```

## 五、数字格式异常（NumberFormatException）

**报错原因**：字符串无法转换为对应的数字类型

**常见场景**：

1. Integer.parseInt () 传入非数字字符串
2. 字符串包含空格、特殊字符、字母
3. 数值超出对应类型的取值范围

**解决方案：**

```java
// 错误示例
int num = Integer.parseInt("123a"); // 包含字母，无法转换

// 正确示例：使用try-catch捕获异常，做合法性校验
public static Integer safeParseInt(String str) {
    if (str == null || str.trim().isEmpty()) {
        return null;
    }
    try {
        return Integer.parseInt(str.trim());
    } catch (NumberFormatException e) {
        System.out.println("数字格式错误：" + str);
        return null;
    }
}
```

## 六、类找不到异常（ClassNotFoundException / NoClassDefFoundError）

### 1. ClassNotFoundException

**报错原因**：编译时类存在，运行时找不到对应的类文件

**常见场景**：

- 依赖包缺失
- 类名全限定名拼写错误
- 反射加载类时类名错误

**解决方案：**

1. 检查 maven/gradle 依赖是否正确引入
2. 核对类的全限定名是否正确
3. 检查编译后的 class 文件是否存在

### 2. NoClassDefFoundError

**报错原因**：编译时能找到类，运行时类的初始化失败 / 依赖缺失

**常见场景**：

- 静态代码块初始化异常
- 类依赖的 jar 包运行时缺失

**解决方案：**

1. 查看异常堆栈，找到类初始化失败的原因
2. 检查运行时环境的依赖包是否完整

## 七、栈溢出异常（StackOverflowError）

**报错原因**：方法调用栈深度超过 JVM 允许的最大栈深度

**常见场景**：

1. 递归方法没有正确的终止条件，无限递归
2. 递归深度过大
3. 方法之间循环调用

**解决方案：**

```java
// 错误示例：无限递归
public void test() {
    test(); // 无终止条件，栈溢出
}

// 正确示例：递归添加终止条件
public int factorial(int n) {
    if (n <= 1) { // 终止条件
        return 1;
    }
    return n * factorial(n-1);
}

// 大深度递归优化：改为循环实现
public int factorialLoop(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}
```

## 八、内存溢出异常（OutOfMemoryError）

**报错原因**：JVM 堆内存不足，无法分配新的对象

**常见场景**：

1. 大对象（如超大数组、长字符串）持续创建
2. 集合中对象未释放，导致内存泄漏
3. 循环中不断创建新对象，无回收

**解决方案：**

1. 优化代码，避免创建不必要的大对象
2. 及时释放无用对象，集合使用完清空
3. 合理设置 JVM 堆内存参数（-Xms、-Xmx）
4. 使用内存分析工具排查内存泄漏点
