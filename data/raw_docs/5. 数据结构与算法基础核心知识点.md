# 数据结构与算法基础核心知识点

## 一、数据结构基础篇

### 1. 线性表

#### 数组（Array） 

- **核心定义**：连续的内存空间，存储相同类型数据的线性结构 
- **核心特点**： 
  -   随机访问：通过下标直接访问元素，时间复杂度O(1)  
  - 插入/删除：需要移动元素，平均时间复杂度O(n)  
  - 固定容量：初始化时确定长度，无法动态扩容（高级语言的动态数组底层是数组复制实现） 
-  **适用场景**：频繁查询、数据量固定的场景

####  链表（Linked List） 

* **核心定义**：通过指针/引用将零散的内存块串联起来的线性结构 
* **常见类型**：单链表、双向链表、循环链表 
* **核心特点**：  
  * 不支持随机访问，遍历查找时间复杂度O(n)  
  * 插入/删除：仅需修改指针，时间复杂度O(1)（找到目标节点后）  
  * 动态容量：无需提前指定长度，内存利用率高 
* **适用场景**：频繁插入删除、数据量动态变化的场景

#### 栈（Stack）

*  **核心定义**：后进先出（LIFO）的线性结构，仅允许在栈顶进行插入和删除操作 -
* **核心操作**：  
  * 入栈push：栈顶添加元素，O(1)  
  * 出栈pop：栈顶移除元素，O(1)  
  * 取栈顶peek：获取栈顶元素，不删除，O(1) 
* **适用场景**：函数调用栈、括号匹配、表达式求值、浏览器前进后退

#### **队列（Queue）**

* **核心定义**：先进先出（FIFO）的线性结构，队尾入队，队头出队 
* **常见类型**：普通队列、循环队列、双端队列、优先级队列 
*  **核心操作**：  
  * 入队enqueue：队尾添加元素，O(1)  
  * 出队dequeue：队头移除元素，O(1) 
*  **适用场景**：任务调度、消息队列、广度优先搜索（BFS）

### 2. 哈希表（Hash Table）

* **核心定义**：通过哈希函数将键（key）映射到数组下标，实现O(1)时间复杂度的查找、插入、删除 
* **核心原理**：  
  *  哈希函数：将key转换为数组下标，要求均匀分布、低碰撞  
  *  哈希冲突：不同key映射到相同下标，解决方案：    
    1. 链地址法：数组每个位置存储链表，冲突元素加入链表    
    2. 开放寻址法：冲突时寻找下一个空位置 
* **核心特点**：  
  * 平均时间复杂度：增删查改均为O(1)  
  * 最坏时间复杂度：O(n)（哈希冲突严重时）  
  * 无序存储 
* **适用场景**：去重、频率统计、缓存、快速查找

###  3. 树（Tree） 

#### 二叉树（Binary Tree）

* **核心定义**：每个节点最多有两个子节点（左子节点、右子节点）的树形结构 -
* **常见类型**：  
  * 满二叉树：所有叶子节点都在最底层，非叶子节点都有两个子节点  
  * 完全二叉树：除了最后一层，其他层节点数都满，最后一层节点靠左排列  
  * 二叉搜索树（BST）：左子树所有节点值 < 根节点值 < 右子树所有节点值，中序遍历有序 
  * 平衡二叉树（AVL）：左右子树高度差不超过1的二叉搜索树，解决BST退化问题 
  * 红黑树：弱平衡的二叉搜索树，插入删除效率高，广泛用于集合、Map实现 
* **核心遍历方式**：  
  1. 前序遍历：根节点 → 左子树 → 右子树  
  2. 中序遍历：左子树 → 根节点 → 右子树 
  3. 后序遍历：左子树 → 右子树 → 根节点  
  4. 层序遍历：按层级从上到下、从左到右遍历（BFS实现） 
* **适用场景**：结构化数据存储、排序、索引、搜索优化

####  堆（Heap）

* **核心定义**：完全二叉树结构，分为大顶堆（根节点值最大）和小顶堆（根节点值最小） 
* **核心操作**：  
  * 插入元素：O(logn)  
  * 取出堆顶元素：O(1)  
  * 删除堆顶元素：O(logn) 
* **适用场景**：优先级队列、TopK问题、堆排序

###  4. 图（Graph） 

- **核心定义**：由顶点（Vertex）和边（Edge）组成的非线性结构 
- **常见分类**：  
  - 无向图 / 有向图  
  - 无权图 / 有权图  
  - 连通图 / 非连通图 
- **存储方式**：  
  1. 邻接矩阵：二维数组，适合稠密图，空间复杂度O(n²)  
  2. 邻接表：数组+链表，适合稀疏图，空间复杂度O(n+e) 
- **核心遍历方式**：  
  1. 深度优先搜索（DFS）：一条路走到黑，再回溯，递归/栈实现  
  2. 广度优先搜索（BFS）：逐层遍历，队列实现  
- **核心算法**：最短路径（Dijkstra、Floyd）、最小生成树（Prim、Kruskal）、拓扑排序 
- **适用场景**：社交网络、地图导航、网络路由、任务依赖调度

## 二、算法基础篇 

### 1. 时间复杂度与空间复杂度 

#### 时间复杂度 

* **定义**：算法执行时间与数据规模的增长关系，用大O表示法，忽略常数项、低阶项 
* **常见复杂度排序（从优到劣）**：  O(1) < O(logn) < O(n) < O(nlogn) < O(n²) < O(2ⁿ) < O(n!) 
*  **核心说明**：  
  * O(1)：常数级，如数组随机访问   
  * O(logn)：对数级，如二分查找  
  * O(n)：线性级，如遍历数组  
  * O(nlogn)：线性对数级，如快速排序、归并排序  
  * O(n²)：平方级，如冒泡排序、插入排序

#### 空间复杂度 

- **定义**：算法运行所需额外内存空间与数据规模的增长关系，同样用大O表示法 
- **核心说明**：  
  - 原地算法：空间复杂度O(1)，不需要额外的大量空间  
  - 递归算法的空间复杂度包含递归调用栈的深度

### 2. 排序算法 

#### 基础排序算法

| 算法     | 平均时间复杂度 | 最坏时间复杂度 | 空间复杂度 | 稳定性 |
| -------- | -------------- | -------------- | ---------- | ------ |
| 冒泡排序 | O(n²)          | O(n²)          | O(1)       | 稳定   |
| 选择排序 | O(n²)          | O(n²)          | O(1)       | 不稳定 |
| 插入排序 | O(n²)          | O(n²)          | O(1)       | 稳定   |
| 希尔排序 | O(n^1.3)       | O(n²)          | O(1)       | 不稳定 |

#### 高效排序算法

| 算法     | 平均时间复杂度 | 最坏时间复杂度 | 空间复杂度 | 稳定性 |
| -------- | -------------- | -------------- | ---------- | ------ |
| 快速排序 | O(nlogn)       | O(n²)          | O(logn)    | 不稳定 |
| 归并排序 | O(nlogn)       | O(nlogn)       | O(n)       | 稳定   |
| 堆排序   | O(nlogn)       | O(nlogn)       | O(1)       | 不稳定 |
| 计数排序 | O(n+k)         | O(n+k)         | O(k)       | 稳定   |
| 桶排序   | O(n+k)         | O(n²)          | O(n+k)     | 稳定   |
| 基数排序 | O(n*k)         | O(n*k)         | O(n+k)     | 稳定   |

**稳定性定义**：相等值的元素，排序后相对位置不变

### 3. 查找算法 

- **顺序查找**：遍历查找，O(n)，无需有序 
- **二分查找**：有序数组中折半查找，O(logn)，效率极高 
- **哈希查找**：通过哈希表查找，平均O(1) 
- **二叉搜索树查找**：平均O(logn)，最坏O(n) 

### 4. 经典算法思想 

#### 贪心算法 

- **核心思想**：每一步都做出当前最优的局部选择，最终希望得到全局最优解 
- **适用场景**：问题具有最优子结构、贪心选择性，如哈夫曼编码、最短路径、最小生成树 
- **特点**：效率高，无法保证全局最优，需证明贪心策略的正确性 

#### 分治算法 

* **核心思想**：将大问题分解为多个规模小的子问题，子问题解决后合并结果得到原问题解 

* **适用场景**：问题可分解、子问题独立、子问题解可合并，如归并排序、快速排序、二分查找 

* **核心步骤**：分解 → 解决 → 合并 

#### 回溯算法 

* **核心思想**：暴力搜索的优化，通过试错的方式，在解空间中搜索，当发现当前路径无法得到解时，回溯回退，尝试其他路径 
* **适用场景**：组合、排列、子集、棋盘、分割等问题，如全排列、N皇后问题 
* **核心框架**：选择 → 递归 → 撤销选择 

#### 动态规划（DP） 

* **核心思想**：将问题分解为重叠子问题，通过存储子问题的解，避免重复计算，最终得到最优解 
* **核心三要素**：最优子结构、重叠子问题、状态转移方程 
* **适用场景**：最值问题、计数问题、存在性问题，如爬楼梯、打家劫舍、最长公共子序列 
* **核心步骤**：定义状态 → 确定状态转移方程 → 初始化边界 → 递推计算 → 提取结果
