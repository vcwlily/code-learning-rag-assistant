# Python 常见面试题与解答

## 1. Python 的列表和元组有什么区别？

| 特性       | 列表（List）                   | 元组（Tuple）                      |
| ---------- | ------------------------------ | ---------------------------------- |
| **可变性** | 可变（可修改、添加、删除元素） | 不可变（创建后无法修改）           |
| **语法**   | 使用方括号 `[]`                | 使用圆括号 `()`                    |
| **性能**   | 略低（动态分配内存）           | 较高（静态内存分配）               |
| **用途**   | 存储需要动态变化的数据         | 存储固定不变的数据（如配置、坐标） |

**示例**：

```python
# 列表
my_list = [1, 2, 3]
my_list[0] = 10  # 合法

# 元组
my_tuple = (1, 2, 3)
# my_tuple[0] = 10  # 报错：TypeError
```

## 2. Python 如何处理异常？try-except 的用法是什么？

Python 通过 `try-except` 语句捕获和处理异常，避免程序崩溃。

**基本语法**：

```python
try:
    # 可能抛出异常的代码
    result = 10 / 0
except ZeroDivisionError:
    # 捕获特定异常并处理
    print("除数不能为零")
except Exception as e:
    # 捕获其他异常
    print(f"发生错误：{e}")
else:
    # 无异常时执行
    print("操作成功")
finally:
    # 无论是否异常都执行
    print("清理资源")
```

## 3. 什么是 Python 的装饰器？怎么使用？

**装饰器**是一种设计模式，用于在不修改原函数代码的情况下，扩展函数的功能（如日志、计时、权限验证）。

**使用方法**：通过 `@装饰器名` 语法应用。

**示例**：

```python
# 定义装饰器
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("函数执行前")
        result = func(*args, **kwargs)
        print("函数执行后")
        return result
    return wrapper

# 应用装饰器
@my_decorator
def say_hello(name):
    print(f"Hello, {name}!")

say_hello("Alice")
```

## 4. Python 的深拷贝和浅拷贝有什么区别？

| 拷贝类型   | 行为                                                         |
| ---------- | ------------------------------------------------------------ |
| **浅拷贝** | 仅复制对象的引用，新对象与原对象共享嵌套对象（如列表中的子列表）。 |
| **深拷贝** | 递归复制对象及其所有嵌套对象，新对象与原对象完全独立。       |

**示例**：

```python
import copy

a = [1, [2, 3]]
b = copy.copy(a)    # 浅拷贝
c = copy.deepcopy(a)# 深拷贝

a[1][0] = 99
print(b)  # [1, [99, 3]]（受影响）
print(c)  # [1, [2, 3]]（不受影响）
```

## 5. 列表推导式的用法和优势是什么？

**用法**：用简洁的语法创建列表，格式为 `[表达式 for 变量 in 可迭代对象 if 条件]`。

**优势**：代码更简洁、可读性强，且执行效率高于普通 `for` 循环。

**示例**：

```python
# 普通写法
squares = []
for i in range(10):
    if i % 2 == 0:
        squares.append(i ** 2)

# 列表推导式
squares = [i ** 2 for i in range(10) if i % 2 == 0]
```

## 6. Python 中的 self 参数有什么作用？

- `self` 是类方法的第一个参数，**指向当前实例对象本身**。
- 用于访问实例的属性和方法，区分实例变量与局部变量。

**示例**：

```python
class Person:
    def __init__(self, name):
        self.name = name  # self.name 是实例属性

    def say_hello(self):
        print(f"我是{self.name}")  # 通过 self 访问属性

p = Person("Bob")
p.say_hello()
```

## 7. 什么是递归函数？递归的优缺点是什么？

**递归函数**：函数内部调用自身的函数，需满足两个条件：

1. **基线条件**：停止递归的条件（避免无限循环）。
2. **递归条件**：函数调用自身的条件。

**优缺点**：

- 优点：代码简洁，逻辑清晰（适合分治问题，如阶乘、斐波那契数列）。
- 缺点：效率低（函数调用开销大），可能导致栈溢出（递归深度过大）。

**示例**：

```python
def factorial(n):
    if n == 1:  # 基线条件
        return 1
    return n * factorial(n - 1)  # 递归条件
```

## 8. Python 的面向对象三大特性是什么？

1. **封装**：将数据和方法封装在类中，通过访问控制（如私有属性 `__attr`）隐藏内部细节，仅暴露必要接口。
2. **继承**：子类继承父类的属性和方法，实现代码复用（支持多继承）。
3. **多态**：不同对象对同一方法的响应不同，通过方法重写实现。

**示例**：

```python
class Animal:
    def speak(self):
        pass

class Dog(Animal):
    def speak(self):
        print("汪汪汪")

class Cat(Animal):
    def speak(self):
        print("喵喵喵")

def make_speak(animal):
    animal.speak()  # 多态：不同对象调用同一方法

make_speak(Dog())
make_speak(Cat())
```

## 9. 如何解决 Python 的索引越界问题？

索引越界会抛出 `IndexError`，常见解决方法：

1. **检查索引范围**：确保索引在 `0 <= index < len(列表)` 内。

2. **使用 `try-except` 捕获异常**：

   ```python
   my_list = [1, 2, 3]
   try:
       print(my_list[5])
   except IndexError:
       print("索引越界")
   ```

3. **使用切片**：切片越界不会报错，而是返回空列表：

   ```python
   print(my_list[5:10])  # 输出 []
   ```

## 10. Python 中的生成器和迭代器有什么区别？

| 特性         | 迭代器（Iterator）                       | 生成器（Generator）                       |
| ------------ | ---------------------------------------- | ----------------------------------------- |
| **定义**     | 实现 `__iter__` 和 `__next__` 方法的对象 | 使用 `yield` 关键字的函数，是迭代器的一种 |
| **创建方式** | 通过 `iter()` 函数或自定义类             | 通过生成器函数或生成器表达式              |
| **内存占用** | 需一次性加载所有数据                     | 按需生成数据，节省内存                    |

**示例**：

```python
# 迭代器
my_iter = iter([1, 2, 3])
print(next(my_iter))  # 1

# 生成器函数
def my_generator():
    yield 1
    yield 2
    yield 3

gen = my_generator()
print(next(gen))  # 1
```

## 11. 简述 Python 的内存管理机制

Python 内存管理以**引用计数**为主，**分代垃圾回收**为辅：

- **引用计数**：每个对象维护一个引用计数，当计数为 0 时对象被销毁。`sys.getrefcount()`可查看计数（注意函数调用会临时 + 1）。
- **分代回收**：将对象分为 0、1、2 三代，新对象在 0 代，经过多次回收仍存活则升入更高代，减少扫描频率。
- **循环引用处理**：通过标记 - 清除算法解决容器对象的循环引用问题。

**示例**：

```python
import sys
a = []
print(sys.getrefcount(a))  # 输出2（a本身+getrefcount临时引用）
b = a
print(sys.getrefcount(a))  # 输出3
del b
print(sys.getrefcount(a))  # 输出2
```

## 12. 什么是 Python 的 GIL（全局解释器锁）？对多线程有何影响？

- **GIL 定义**：CPython 解释器中的一把互斥锁，确保同一时刻只有一个线程在 CPU 上执行字节码，避免解释器状态的并发冲突。

- 对多线程的影响

  ：

  - 多线程无法利用多核 CPU 实现真正的并行（仅能并发），因为 GIL 会在线程切换时释放和获取。
  - **IO 密集型任务**（如网络请求、文件读写）仍适合多线程，因为 IO 等待时 GIL 会释放。
  - **CPU 密集型任务**建议用多进程，每个进程有独立的 GIL，可利用多核。

## 13. 什么是闭包（Closure）？Python 中如何实现？

**闭包**是指能够记住外层函数作用域变量的内层函数，即使外层函数已执行完毕。需满足三个条件：

1. 存在嵌套函数（内层函数定义在外层函数内）。
2. 内层函数引用外层函数的变量。
3. 外层函数返回内层函数对象。

**示例**：

```python
def outer(x):
    def inner(y):
        return x + y  # 引用外层变量x
    return inner

add_5 = outer(5)
print(add_5(3))  # 输出8（记住了x=5）
```

## 14. Python 中多进程和多线程的区别及适用场景？

| 特性         | 多进程（Multiprocessing）          | 多线程（Threading）               |
| ------------ | ---------------------------------- | --------------------------------- |
| **内存空间** | 独立内存空间（数据隔离）           | 共享内存空间（需同步）            |
| **GIL 影响** | 无（每个进程有独立 GIL）           | 有（无法多核并行）                |
| **开销**     | 大（进程创建 / 切换成本高）        | 小（线程轻量）                    |
| **适用场景** | CPU 密集型任务（如计算、图像处理） | IO 密集型任务（如爬虫、Web 服务） |

**多进程示例**：

```python
from multiprocessing import Process

def task(name):
    print(f"进程 {name} 执行中")

if __name__ == "__main__":
    p = Process(target=task, args=("A",))
    p.start()
    p.join()
```

## 15. 解释 Python 的 “鸭子类型”（Duck Typing）

鸭子类型是 Python 动态类型的核心思想：**“如果一个对象走起来像鸭子、叫起来像鸭子，那它就是鸭子”**。

即不关注对象的具体类型，只关注对象是否具有所需的方法或属性。

**示例**：

```python
class Duck:
    def quack(self):
        print("嘎嘎嘎")

class Person:
    def quack(self):
        print("我也会学鸭子叫")

def make_quack(obj):
    obj.quack()  # 不检查类型，只要有quack方法即可

make_quack(Duck())    # 输出“嘎嘎嘎”
make_quack(Person())  # 输出“我也会学鸭子叫”
```

## 16. 如何在 Python 中实现单例模式？（至少两种方法）

单例模式确保一个类只有一个实例，常见实现方式：

### 方法 1：使用`__new__`方法

```python
class Singleton:
    _instance = None
    def __new__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super().__new__(cls)
        return cls._instance

s1 = Singleton()
s2 = Singleton()
print(s1 is s2)  # 输出True
```

### 方法 2：使用装饰器

```python
def singleton(cls):
    instances = {}
    def wrapper(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]
    return wrapper

@singleton
class MyClass:
    pass

m1 = MyClass()
m2 = MyClass()
print(m1 is m2)  # 输出True
```

## 17. Python 中`*args`和`**kwargs`的作用是什么？

- `*args`：用于接收**任意数量的位置参数**，打包成元组（tuple）。
- `**kwargs`：用于接收**任意数量的关键字参数**，打包成字典（dict）。

**示例**：

```python
def func(*args, **kwargs):
    print("位置参数：", args)
    print("关键字参数：", kwargs)

func(1, 2, 3, name="Alice", age=20)
# 输出：
# 位置参数： (1, 2, 3)
# 关键字参数： {'name': 'Alice', 'age': 20}
```

## 18. 解释 Python 的模块（Module）和包（Package）的区别

- **模块**：一个`.py`文件就是一个模块，包含函数、类、变量等，可通过`import`导入。
- **包**：包含`__init__.py`文件的目录，用于组织多个相关模块，避免命名冲突。`__init__.py`可初始化包或控制导入内容。

**导入示例**：

```python
# 导入模块
import math
print(math.pi)

# 导入包中的模块
from my_package import my_module
my_module.my_function()
```

## 19. 什么是 Python 的上下文管理器（Context Manager）？如何使用？

上下文管理器用于**自动管理资源**（如文件、数据库连接），确保资源在使用后被正确释放，通过`with`语句使用。需实现`__enter__`和`__exit__`两个方法：

- `__enter__`：进入`with`块时执行，返回资源对象。
- `__exit__`：离开`with`块时执行（无论是否异常），负责释放资源。

**示例（文件操作）**：

```python
# 自动关闭文件，无需手动f.close()
with open("test.txt", "r") as f:
    content = f.read()
```

**自定义上下文管理器**：

```python
class MyContext:
    def __enter__(self):
        print("进入上下文")
        return "资源对象"
    def __exit__(self, exc_type, exc_val, exc_tb):
        print("离开上下文，释放资源")

with MyContext() as obj:
    print("使用", obj)
```

## 20. 简述 Python 字典（dict）的实现原理

Python 字典基于 ** 哈希表（Hash Table）** 实现，核心特点：

1. **哈希函数**：通过键（key）的哈希值计算存储位置，时间复杂度 O (1)。
2. **哈希冲突处理**：采用**开放寻址法**（线性探测），当位置被占用时，寻找下一个空闲位置。
3. **动态扩容**：当哈希表负载因子（已用位置 / 总位置）超过 2/3 时，自动扩容为原大小的 2 倍，重新哈希所有键值对。

**优势**：查找、插入、删除操作平均时间复杂度为 O (1)，是 Python 中高效的键值对存储结构。
