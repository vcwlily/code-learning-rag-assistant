# 《代码学习智能审查助手》技术选型对比文档
## 一、项目背景
本项目是基于RAG与大模型的编程答疑系统，核心面向编程学习者，提供代码分析、智能答疑能力，技术选型核心围绕「开发效率高、适配AI场景、复试可讲性强、轻量易部署」四个核心原则。

## 二、核心技术选型对比
### 1. 后端框架选型：FastAPI vs Flask
| 对比维度 | FastAPI | Flask | 最终选型 |
|----------|---------|-------|----------|
| 性能 | 基于异步Starlette框架，并发性能优异，适配AI接口高并发场景 | 同步框架，并发性能弱，高并发场景需额外改造 | FastAPI |
| 开发效率 | 原生支持类型提示，自动参数校验，自动生成OpenAPI接口文档，减少重复代码 | 手动编写参数校验、接口文档，开发效率低 | FastAPI |
| 生态适配 | 与LangChain、大模型工具链原生适配，AI应用开发生态完善 | 通用Web框架，AI场景适配需额外开发 | FastAPI |
| 学习成本 | 基于Python标准，上手简单，文档完善 | 上手简单，但高阶功能需额外学习 | 持平 |

选型理由：FastAPI的异步性能、自动接口文档能力，更适配AI应用的开发场景，同时在复试中能体现对现代Web框架的理解，拉开与Flask的同质化差距。

### 2. 向量数据库选型：ChromaDB vs FAISS
| 对比维度 | ChromaDB | FAISS | 最终选型 |
|----------|----------|-------|----------|
| 部署难度 | 轻量本地持久化部署，单文件存储，无需额外服务 | 仅内存索引，持久化需额外开发，部署复杂 | ChromaDB |
| 功能完整性 | 原生支持元数据过滤、文档增删改查，适配RAG全流程 | 仅核心向量检索能力，周边功能需自行开发 | ChromaDB |
| 性能 | 轻量场景下性能优异，满足万级以内文档检索需求 | 海量数据场景性能更强，本项目用不到 | 持平 |
| 开发效率 | Python SDK完善，3行代码即可完成向量库创建 | 需自行封装索引、持久化逻辑，开发成本高 | ChromaDB |

选型理由：ChromaDB轻量易部署，原生支持持久化，完全满足本项目的知识库规模需求，开发效率高，复试中能完整讲解RAG的向量存储全流程。

### 3. 大模型API选型：DeepSeek API vs 通义千问/文心一言
| 对比维度 | DeepSeek API | 通义千问/文心一言 | 最终选型 |
|----------|--------------|--------------------|----------|
| 代码能力 | 原生针对代码场景优化，代码理解、纠错、解释能力优异 | 通用场景能力强，代码垂直场景能力弱于DeepSeek | DeepSeek |
| 调用成本 | 输入成本极低，百万token仅需1元，全项目开发成本不超过5元 | 调用成本更高，同等token量成本是DeepSeek的3-5倍 | DeepSeek |
| 长上下文支持 | 原生支持128K长上下文，适配长代码文件分析场景 | 长上下文需更高版本接口，成本更高 | DeepSeek |
| 开发适配 | API完全兼容OpenAI格式，SDK简单，上手快 | 接口格式有差异，需额外适配 | DeepSeek |

选型理由：DeepSeek在代码垂直场景的能力更强，调用成本更低，完全适配本项目的编程答疑场景，复试中能体现对垂直领域大模型选型的思考。

### 4. 前端框架选型：Streamlit vs Gradio
| 对比维度 | Streamlit | Gradio | 最终选型 |
|----------|-----------|--------|----------|
| 组件丰富度 | 原生支持代码编辑器、多栏布局、会话状态管理，组件更丰富 | 核心面向AI演示场景，组件简单，自定义能力弱 | Streamlit |
| 交互体验 | 支持页面无刷新更新、历史对话管理，交互体验更好 | 交互逻辑简单，复杂场景适配性差 | Streamlit |
| 部署难度 | 官方提供免费社区云部署，一键部署，无需服务器 | 部署需额外配置，免费部署链路更复杂 | Streamlit |
| 学习成本 | 纯Python开发，无需前端知识，上手极快 | 同样纯Python开发，上手简单 | 持平 |

选型理由：Streamlit原生支持代码编辑器，完全适配本项目的代码输入场景，部署简单，能快速完成可视化交互界面开发，复试中能直接给面试官演示在线项目。

## 三、最终技术栈
后端框架：FastAPI
向量数据库：ChromaDB
大模型能力：DeepSeek API
前端框架：Streamlit
RAG工具链：LangChain
检索优化：Cohere Re-rank 重排序模型
