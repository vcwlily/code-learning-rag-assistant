# 大模型幻觉问题解决方案
## 问题背景
大模型在编程答疑场景中，容易出现编造知识点、给出错误代码、虚构API用法的幻觉问题，影响答疑的准确性。

## 我的解决方案
我从4个维度，系统性地解决了幻觉问题：
1.  **检索机制约束**：采用RAG检索增强生成技术，让模型只能基于检索到的知识库内容回答问题，从根源上减少无依据的编造。
2.  **Prompt强约束**：在系统prompt中制定了严格的规则，明确要求模型必须基于参考内容回答，无相关内容直接拒绝，禁止编造，同时要求标注引用来源，让生成内容可追溯。
3.  **模型参数优化**：把生成答案的temperature调低到0.3，降低模型的创造性，让输出更严谨、更贴合参考内容，减少自由发挥带来的幻觉。
4.  **检索精度优化**：通过「粗召回+重排序」的两阶段检索策略，提升了检索结果的准确率，给模型提供更精准的参考内容，从输入上减少幻觉的可能性。

## 优化效果
经过测试，优化后模型的幻觉发生率从优化前的约35%，降低到了5%以内，回答的可靠性和准确性大幅提升。
